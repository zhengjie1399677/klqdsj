# 库伦旗农牧产业园大数据平台 - 接口与数据修改指南

本文档专门为了开发与维护人员编写，说明如何在此大屏项目中**修改数据参数**、**替换静态模拟数据**，以及**如何接入真实的后端 API 接口**。

---

## 1. 当前的数据在哪里？

在开发阶段，为了展示大屏的视觉和动态效果，所有的图表数据都是**静态模拟数据 (Mock Data)**。
数据主要分布在以下两个位置：

1. **统一管理的模拟数据总文件**：`src/data/mockData.ts` (大部分图表的数据都集中在这个文件中)
2. **个别组件内部的状态定义**：如 `src/components/ProductionData.tsx`

---

## 2. 各个面板怎样手动修改参数？

### 🏀 生产数据统计 (左上角动态水球)
- **文件位置**：`src/components/ProductionData.tsx`
- **参数说明**：数据直接在这个组件内部的 `useState` 里定义。
- **如何修改**：找到最前面的 `stats` 状态，直接修改 `value` (当前值) 和 `target` (目标值)。水球的水位百分比是系统通过 `(value / target) * 100` 自动计算并限制在100%之内的。
```typescript
const [stats, setStats] = useState({
  today: { value: 1250, target: 2200 },       // 今日生产 (value: 实际值, target: 目标值)
  yesterday: { value: 1420, target: 2800 },   // 昨日生产
  thisMonth: { value: 28500, target: 45000 }, // 本月累计
  lastMonth: { value: 42000, target: 80000 }, // 上月累计
});
```

### 📊 原材料统计看板 (左中带滑动动画的列表)
- **文件位置**：`src/components/RawMaterialStats.tsx` 读取了 `src/data/mockData.ts`
- **参数说明**：找到 `mockData.ts` 中的 `rawMaterialInventoryStats` 数组。
- **如何修改**：直接增、删或修改数组内的对象，可以添加无限多的新种类进去，前端会自动处理滚动播放。
```typescript
export const rawMaterialInventoryStats = [
  { name: '稻草', inbound: 3200, outbound: 3000, inventory: 1100 },
  // name: 材料名称
  // inbound: 入库量
  // outbound: 出库量
  // inventory: 结余库存量
  // 按照这个格式往下添加即可...
];
```

### 📈 其他图表 (折线图、柱状图、饼图等)
- **文件位置**：全都在 `src/data/mockData.ts` 内统一管理。
- **如何修改**：去这个文件中找到对应的数组即可。例如：
  - `productionStatsData` -> 对应右上的生产趋势
  - `orderStatsData` -> 对应右中的订单量
  - `salesStatsData` -> 对应右下的销售统计
  直接修改里面对应的维度字段大小，保存后图表会立即发生变化。

## 3. 🚀 如何接入真实的后端 API 接口？

当你的服务后端（Java / Python / Node 等）开发好数据接口后，你需要将前端目前的静态数据替换为使用 `fetch` 或 `axios` 获取的动态数据。

### 标准接入模板 (针对大屏轮询刷新的场景)

以 **“生产数据统计” (ProductionData.tsx)** 为例：

#### 第一步：清空固定的初始值
将原来的模拟数据清空或置为 0，防止刚进页面时闪烁假数据。
```tsx
const [stats, setStats] = useState({
  today: { value: 0, target: 100 },
  yesterday: { value: 0, target: 100 },
  thisMonth: { value: 0, target: 100 },
  lastMonth: { value: 0, target: 100 },
});
```

#### 第二步：使用 `useEffect` 挂载网络请求并开启定时轮询
把原本写在 `useEffect` 里用来模拟随机波动的代码删掉，替换为你自己的数据请求逻辑：

```tsx
import { useEffect } from 'react';

useEffect(() => {
  // 定义请求后端 API 的函数
  const fetchData = async () => {
    try {
      // 请替换成你真实的 API 接口地址
      const response = await fetch('http://你的服务器IP或域名/api/production/stats');
      const data = await response.json();
      
      // 假设接口返回的 JSON 格式也是带有 today / yesterday 这种结构
      // 直接把查到的数据放进状态里：
      setStats({
        today: data.today,
        yesterday: data.yesterday,
        thisMonth: data.thisMonth,
        lastMonth: data.lastMonth
      });
    } catch (error) {
      console.error("数据加载失败:", error);
    }
  };

  // 1. 组件刚渲染时，马上请求一次最新数据
  fetchData();

  // 2. 大屏特性：开启定时器，每 60 秒轮询一次后端，实现"实时数据"更新
  const intervalId = setInterval(fetchData, 60000); 

  // 3. 必须清理定时器，防止内存溢出
  return () => clearInterval(intervalId);
}, []);
```

### 对于复杂滚动列表的对接提示 (如原材料列表)

在 `RawMaterialStats.tsx` 中，由于它自带一套数组不断 `pop` 和 `unshift` （尾部移出，头部插入）的滚动逻辑：
你只需要用一个**单独的 `useEffect`** 把请求到的后台数组扔给 `setData(真实数据)`，原先管着滑动的 `useEffect` **不需要去动它**，它会自动以新的数组开始轮播。

## 4. 后端接口数据结构封装建议

为了最大限度减少前端组件的修改，你的后端给大屏返回的 JSON 数据字段名，如果是下面的形式是最好的（跟现在的 `mockData.ts` 结构对齐）：

- **图表类（按时间轴）**后端返回例子：
  ```json
  [
    { "name": "周一", "v1": 120, "v2": 80 },
    { "name": "周二", "v1": 132, "v2": 90 }
  ]
  ```
- **占比饼图类**后端返回例子：
  ```json
  [
    { "name": "玉米", "value": 45 },
    { "name": "豆粕", "value": 30 }
  ]
  ```

> 只要后端的 key name 能跟前端现在用的对上，接入真实接口的工作就会只需要加几行 fetch 请求即可，无须调整 UI 代码。
